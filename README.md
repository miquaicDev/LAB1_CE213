# HDL-Lab-Assignments
Answers for Lab 1 of the CE213 Digital System Design with HDL course - UIT
# Verilog HDL Practice – ModelSim

This repository contains two basic **Verilog HDL design exercises** simulated using **ModelSim**.

The goal of these exercises is to practice **RTL design and simulation using testbenches**.

---

# Exercise 1 – Counter Design

## Description

Design a **counter circuit** using Verilog HDL.

The counter has the ability to **load an initial value using Preset and Clear signals** during simulation.

## Requirements

1. Create a new project in ModelSim.
2. Design the counter using Verilog HDL.
3. The initial value of the counter is loaded through **Preset** and **Clear** signals in ModelSim.
4. Write a **testbench** to generate input signals and verify the functionality of the design.

## Files

```
counter.v
counter_tb.v
```

## Simulation

Example simulation steps in ModelSim:

```
vlog counter.v
vlog counter_tb.v
vsim counter_tb
run -all
```

# Exercise 2 – 32×32 Register File

## Description

Design a **Register File containing 32 registers**, each register storing **4 bytes (32 bits)**.

The register file supports:

* **2 read ports**
* **1 write port**

## Signals

| Signal       | Width | Description                          |
| ------------ | ----- | ------------------------------------ |
| ReadAddress1 | 5     | Address of register to read (port 1) |
| ReadAddress2 | 5     | Address of register to read (port 2) |
| WriteAddress | 5     | Address of register to write         |
| WriteData    | 32    | Data to be written                   |
| ReadData1    | 32    | Output data from read port 1         |
| ReadData2    | 32    | Output data from read port 2         |
| ReadWriteEn  | 1     | Write enable signal                  |

## Requirements

1. Create a new project in ModelSim.
2. Implement the **32×32 register file using Verilog HDL**.
3. Write a **testbench** to generate input signals and verify the design through simulation.

## Files

```
rf32x32.v
rf32x32_tb.v
```

## Simulation

```
vlog rf32x32.v
vlog rf32x32_tb.v
vsim rf32x32_tb
run -all
```
